项目开发流程

  1. 确定开发需求(将客户的需求转换为我们自己的需求)出需求文档

  2. 原型图(由UI出)

  3. 技术选型（确定项目中所需要的具体的技术node vue和使用到的一些框架和库，约定规范）

  4. 分配任务开发项目

  5. 上线前的测试和bug修复

  6. 项目上线，客户验收

  7. 前端开发注意事项

    • 前端布局应适应各主流屏幕 (媒体查询, 百分比)
      • 标签的使用需注意 语义化
    • 需兼容主流浏览器/主流操作系统

      • 应考虑用户习惯/方便用户进行操作

    • 团体开发
      • 统一开发工具 换行方式 编码格式
      • 代码冲突需协商后合并
      • 一小改 一commit
      • id 及 class 命名约定

css笔记

  1. css 介绍

    • 作用

      • 设置HTML页面中的文本内容（字体、大小、对齐方式等）、图片的外形（宽高、边框样式、边距等）以及版面的布局等外观显示样式

    • 名字

      • CSS样式表
      • 层叠样式表
      • 级联样式表

    • 使用方式

      • 行内样式

        • <标签名 style="属性1:属性值1; 属性2:属性值2; 属性3:属性值3;"> 内容 </标签名>
        • 书写方便，权重高,控制范围低，不方便维护

      • 内嵌样式

        • 部分结构和样式相分离

      • 外链样式

        • 完全实现结构和样式相分离

  2. 选择器

    • 选择器 与 { 之间必须包含空格。
    • 通配符选择器
      • *
      • 匹配页面中所有的元素
    • id选择器
      • #
      • 元素的id值是唯一的 (相当于身份证)
    • 类选择器
      • .
      • 相当于人的名字
    • 标签选择器
    • 后代选择器 .red .blue
    • 子代选择器 .red>.blue
    • 并集选择器 .red , .blue
    • 伪类选择器
      • :link 未访问的链接
      • :visited 已访问的链接
      • :hover 鼠标移动到链接上
      • :active 选定的链接
      • :link :visited只对a标签起作用
    • 伪元素选择器
      • ::first-letter
        • 文本的第一个单词或字
      • ::first-line
        • 文本第一行
      • ::selection
        • 可改变选中文本的样式；
      • ::before
      • ::after
      • ::before和::after一定要配合content一起使用
    • 结构(位置)伪类选择器
      • :nth-child(n)
        • 匹配属于其父元素的第 N 个子元素，不论元素的类型
      • :first-child
        • 选取属于其父元素的首个子元素的指定选择器
      • :last-child
        • 选取属于其父元素的最后一个子元素的指定选择器
      • :nth-last-child(n)
        • 选择器匹配属于其元素的第 N 个子元素的每个元素，不论元素的类型，从最后一个子元素开始计数。
      • n 可以是数字、关键词或公式
    • 目标伪类选择器
      • :target
      • 改变目标的样式
    • 属性选择器 input[type="password"]
    • 交集选择器 p.red
      • 交集选择器中如果有标签选择器，标签选择器一定要写在最前面

  3. class和id的命名规则

    • 1.可以包含字母、数字、下划线、短横线
    • 2.不能以数字开头
    • 3.不能以短横线开头
    • 4.不能以单个短横线
    • 5.长名称或词组可以使用中横线来为选择器命名
    • 6.不建议使用“_”下划线来命名CSS选择器。
    • 建议：
      • 1.使用有意义的单词
      • 2.使用驼峰命名法。oneTwo userName

  4. 元素默认样式

    • margin 定义外边距
    • padding 定义内边距
    • outline
    • border

  12. 背景

    • background-color
    • background-image: url(./img/亚索.gif)
    • background-repeat: no-repeat;
      • repeat : 　背景图像在纵向和横向上平铺（默认的）
      • no-repeat : 　背景图像不平铺
      • repeat-x : 　背景图像在横向上平铺
      • repeat-y : 　背景图像在纵向平铺
    • background-position: 10px 20px;
      • top | center | bottom | left | center | right
    • background-attachment: fixed
      • scroll : 　背景图像是随对象内容滚动
      • fixed : 　背景图像固定
    • background-size
      • 可以设置长度单位(px)或百分比（设置百分比时，参照盒子的宽高）
      • 设置为cover时，会自动调整缩放比例，保证图片始终填充满背景区域，如有溢出部分则会被隐藏。我们平时用的cover 最多
      • 设置为contain会自动调整缩放比例，保证图片始终完整显示在背景区域。
    • 背景连写：background:background-image background-repeat background-position;
      • background-image一定不能少，其他的可选，而且与顺序无关
    • 多背景
      • background:背景颜色 背景图片地址 背景平铺 背景滚动 背景位置
      • background: url(./images/lock.png) no-repeat 200px 0,url(./images/phone.png) no-repeat,url(./images/user_fwfw.png) no-repeat,blue;
      • 如果设置多背景的时候，还需要设置背景颜色，背景颜色一定要放在最后面

  13. 边框 border

  14. 定位 position

    • 1.静态定位
      • static
      • 遵循HTML的规则，所有元素的默认定位就是static
    • 2.相对定位
      • relative
      • 1.不脱标，占原来的位置
      • 2.相对与自身原来在标准流中的位置来做位置移动
      • 3.不能改变元素的显示方式。
      • 4.如果只设置了定位方式，而没有设置偏移量，那么元素的位置不变
    • 3.绝对定位
      • absolute
      • 1.脱标，不占原来的位置
      • 2.可以改变元素的显示方式
      • 3.如果有包含关系：父辈元素都是静态定位，那么这个绝对定位的子元素以浏览器为参考标准来做位置的移动，如果父辈元素中有非静态定位的元素，那么这个绝对定位的子元素以这个非静态定位的父辈元素为参考来做位置的移动，如果父辈元素中都是给静态定位的元素，那么这个绝对定位的子元素遵循就近原则。
      • 4.如果只设置了定位方式，而没有设置偏移量，那么元素的位置不变
      • 子绝父相
    • 4.固定定位
      • fixed
      • 1.脱标，不占位置
      • 2.如果这个元素固定定位，不管父辈元素是什么定位，这个固定定位的元素都是以浏览器为参考来做位置的移动

  15. 层级

  16. 浮动 float

    • 浮动的特点：
      • 1.浮动的元素脱离标准流，不占原来的位置
      • 2.浮动的元素显示方式被改变
      • 3.浮动的元素以顶部对齐
      • 4.如果浮动的元素前面是标准流的元素，那么这个浮动的元素只会跟在标准流元素的后面,除非通过人为去改变才能让他的位置发生变化
      • 5.当有包含关系的时候，如果子元素浮动，不会跑出父元素，除非人为改动
    • 清除浮动的方式：
      • 1.使用clear
      • 2.有包含关系，给父元素加高度
      • 3.给父盒子加overflow：hidden；
      • 4.额外标签法
      • 5.伪元素清除浮动(这是开发中最常用，这个必须记住，而且要手写出这段代码)

常见的网络传输数据格式

  1. xml

xml是一种最早的网络传输协议，常见于java web开发中，不单单作为网络层的参数协议，还常见于各种配置文件中，在移动开发中也常见但是已不是主流的网络传输协议。
优点：可读性强，解析方便；
缺点：效率不高，资源消耗过大；
解析方式：DOM解析，SAX解析，PULL解析；

（1）DOM解析：
解析器读入整个文档，然后构建一个驻留内存的树结构，然后代码就可以使用 DOM 接口来操作这个树结构。优点：整个文档树在内存中，便于操作；支持删除、修改、重新排列等多种功能；缺点：将整个文档调入内存（包括无用的节点），浪费时间和空间；使用场合：一旦解析了文档还需多次访问这些数据；硬件资源充足（内存、CPU）
（2）SAX解析：
SAX ，事件驱动型解析方式。当解析器发现元素开始、元素结束、文本、文档的开始或结束等时，发送事件，程序员编写响应这些事件的代码，保存数据。优点：不用事先调入整个文档，占用资源少；SAX解析器代码比DOM解析器代码小，适于Applet，下载。缺点：不是持久的；事件过后，若没保存数据，那么数据就丢了；无状态性；从事件中只能得到文本，但不知该文本属于哪个元素；使用场合：Applet;只需XML文档的少量内容，很少回头访问；机器内存少；
（3）PULL解析：
PULL解析方式是Android专门为移动设备上解析XML文件而设计的一种解析方式，显而易见的其更加适用于移动设备解析xml文件。Pull解析和Sax解析很相似,Pull解析和Sax解析不一样的地方是pull读取xml文件后触发相应的事件调用方法返回的是数字还有pull可以在程序中控制想解析到哪里就可以停止解析。

  1. json
    SON是在移动端比较常见的网络传输协议，它较xml格式更叫的简单和“小”，因此比xml更适合移动端对流量和内存的控制。

优点：较XML格式更加小巧；
缺点：传输效率也不是太别高，但相较于xml提高了很多；
解析方式：Gson解析，JSONObject方式解析，FastJson解析

（1）Gson解析：
Gson解析方式是Google开源的一套解析方式，通过提供的Gson jar包，通过静态方法直接由字符串解析成java对象，简单方便。
具体使用方法，可参考：Google Gson 使用简介

（2）JSONObject解析：
JSONObject在org.json下面的包中，其也是一个解析Json字符串的工具类，具体使用方式可参考：JSONObject与JSONArray的使用

（3）FastJson解析：
FastJson是阿里巴巴开源的一个解析Json数据的类库，能够将json字符串解析成java对象

  1. protobuf

ProtoBuf是Google开源的一套二进制流网络传输协议，它独立于语言，独立于平台。google 提供了多种语言的实现：java、c#、c++、go 和 python，每一种实现都包含了相应语言的编译器以及库文件。由于它是一种二进制的格式，比使用 xml 进行数据交换快许多。可以把它用于分布式应用之间的数据通信或者异构环境下的数据交换。作为一种效率和兼容性都很优秀的二进制数据传输格式，可以用于诸如网络传输、配置文件、数据存储等诸多领域。

优点：传输效率快（比xml和json快10-20倍），文档型协议；
缺点：使用不太方便

    • 主要目的就是让大家直观的认识标签(markup)和属性(attribute)的用途和作用
    • 使用有意义的单词来作为标签的名字
    • 核心：在合适的地方使用合适的标签
    • 优点
      • 语义化HTML会使HTML结构变的清晰，有利于维护代码和添加样式
      • 让浏览器或是网络爬虫可以很好地解析，从而更好分析其中的内容
      • 便于团队开发和维护，语义化更具可读性，是下一步吧网页的重要动向，遵循W3C标准的团队都遵循这个标准，可以减少差异化。
    • 语义是否良好?
      • 当我们去掉CSS之后，网页结构依然组织有序，并且有良好的可读性

  9. HTML5全局属性

    • data-* 使用 data-* 属性来嵌入自定义数据。
      • 属性名不应该包含任何大写字母，并且在前缀 “data-“ 之后必须有至少一个字符,属性值可以是任意字符串。
    • hidden
      • 它规定元素仍未或不再相关。浏览器不应显示已规定 hidden 属性的元素。hidden 属性也可用于防止用户查看元素，直到匹配某些条件（比如选择了某个复选框）。然后，JavaScript 可以删除 hidden 属性，以使此元素可见。
    • spellcheck
      • 规定是否对元素进行拼写和语法检查。
    • tabindex
      • abindex 属性规定元素的 tab 键控制次序（当 tab 键用于导航时）
      • 以下元素支持 tabindex 属性：<a>, <area>, <button>, <input>, <object>, <select> 以及 <textarea>。
    • contenteditable
      • 规定元素内容是否可编辑。
      • 可继承
