css笔记

  1. css 介绍

    • 作用

      • 设置HTML页面中的文本内容（字体、大小、对齐方式等）、图片的外形（宽高、边框样式、边距等）以及版面的布局等外观显示样式

    • 名字

      • CSS样式表
      • 层叠样式表
      • 级联样式表

    • 使用方式

      • 行内样式

        • <标签名 style="属性1:属性值1; 属性2:属性值2; 属性3:属性值3;"> 内容 </标签名>
        • 书写方便，权重高,控制范围低，不方便维护

      • 内嵌样式

        • 部分结构和样式相分离

      • 外链样式

        • 完全实现结构和样式相分离

  2. 选择器

    • 选择器 与 { 之间必须包含空格。
    • 通配符选择器
      • *
      • 匹配页面中所有的元素
    • id选择器
      • #
      • 元素的id值是唯一的 (相当于身份证)
    • 类选择器
      • .
      • 相当于人的名字
    • 标签选择器
    • 后代选择器 .red .blue
    • 子代选择器 .red>.blue
    • 并集选择器 .red , .blue
    • 伪类选择器
      • :link 未访问的链接
      • :visited 已访问的链接
      • :hover 鼠标移动到链接上
      • :active 选定的链接
      • :link :visited只对a标签起作用
    • 伪元素选择器
      • ::first-letter
        • 文本的第一个单词或字
      • ::first-line
        • 文本第一行
      • ::selection
        • 可改变选中文本的样式；
      • ::before
      • ::after
      • ::before和::after一定要配合content一起使用
    • 结构(位置)伪类选择器
      • :nth-child(n)
        • 匹配属于其父元素的第 N 个子元素，不论元素的类型
      • :first-child
        • 选取属于其父元素的首个子元素的指定选择器
      • :last-child
        • 选取属于其父元素的最后一个子元素的指定选择器
      • :nth-last-child(n)
        • 选择器匹配属于其元素的第 N 个子元素的每个元素，不论元素的类型，从最后一个子元素开始计数。
      • n 可以是数字、关键词或公式
    • 目标伪类选择器
      • :target
      • 改变目标的样式
    • 属性选择器 input[type="password"]
    • 交集选择器 p.red
      • 交集选择器中如果有标签选择器，标签选择器一定要写在最前面

  3. class和id的命名规则

      • 1.使用有意义的单词
      • 2.使用驼峰命名法。oneTwo userName

  12. 背景

    • background-color
    • background-image: url(./img/亚索.gif)
    • background-repeat: no-repeat;
      • repeat : 　背景图像在纵向和横向上平铺（默认的）
      • no-repeat : 　背景图像不平铺
      • repeat-x : 　背景图像在横向上平铺
      • repeat-y : 　背景图像在纵向平铺
    • background-position: 10px 20px;
      • top | center | bottom | left | center | right
    • background-attachment: fixed
      • scroll : 　背景图像是随对象内容滚动
      • fixed : 　背景图像固定
    • background-size
      • 可以设置长度单位(px)或百分比（设置百分比时，参照盒子的宽高）
      • 设置为cover时，会自动调整缩放比例，保证图片始终填充满背景区域，如有溢出部分则会被隐藏。我们平时用的cover 最多
      • 设置为contain会自动调整缩放比例，保证图片始终完整显示在背景区域。
    • 背景连写：background:background-image background-repeat background-position;
      • background-image一定不能少，其他的可选，而且与顺序无关
    • 多背景
      • background:背景颜色 背景图片地址 背景平铺 背景滚动 背景位置
      • background: url(./images/lock.png) no-repeat 200px 0,url(./images/phone.png) no-repeat,url(./images/user_fwfw.png) no-repeat,blue;
      • 如果设置多背景的时候，还需要设置背景颜色，背景颜色一定要放在最后面

  14. 定位 position

    • 1.静态定位
      • static
      • 遵循HTML的规则，所有元素的默认定位就是static
    • 2.相对定位
      • relative
      • 1.不脱标，占原来的位置
      • 2.相对与自身原来在标准流中的位置来做位置移动
      • 3.不能改变元素的显示方式。
      • 4.如果只设置了定位方式，而没有设置偏移量，那么元素的位置不变
    • 3.绝对定位
      • absolute
      • 1.脱标，不占原来的位置
      • 2.可以改变元素的显示方式
      • 3.如果有包含关系：父辈元素都是静态定位，那么这个绝对定位的子元素以浏览器为参考标准来做位置的移动，如果父辈元素中有非静态定位的元素，那么这个绝对定位的子元素以这个非静态定位的父辈元素为参考来做位置的移动，如果父辈元素中都是给静态定位的元素，那么这个绝对定位的子元素遵循就近原则。
      • 4.如果只设置了定位方式，而没有设置偏移量，那么元素的位置不变
      • 子绝父相
    • 4.固定定位
      • fixed
      • 1.脱标，不占位置
      • 2.如果这个元素固定定位，不管父辈元素是什么定位，这个固定定位的元素都是以浏览器为参考来做位置的移动

  16. 浮动 float

    • 浮动的特点：
      • 1.浮动的元素脱离标准流，不占原来的位置
      • 2.浮动的元素显示方式被改变
      • 3.浮动的元素以顶部对齐
      • 4.如果浮动的元素前面是标准流的元素，那么这个浮动的元素只会跟在标准流元素的后面,除非通过人为去改变才能让他的位置发生变化
      • 5.当有包含关系的时候，如果子元素浮动，不会跑出父元素，除非人为改动
    • 清除浮动的方式：
      • 1.使用clear属性
      • 2.有包含关系，给父元素加高度
      • 3.给父盒子加overflow：hidden；
      • 4.额外标签法
      • 5.伪元素清除浮动(这是开发中最常用，这个必须记住，而且要手写出这段代码)

  31. 元素的显示方式

    • display: none
    • visibility: hidden;
    • 1.块级元素 display:block
      • 1.独占一行
      • 2.可以设置有效宽高
      • 常见的块级元素：
        • div p h1 h2 h3 h4 h5 h6 ul li ol dl dd dt
    • 2.行内元素 display:inline
      • 1.一行可以显示多个
      • 2.不能设置有效宽高
      • 常见的行内元素：
        • span a em strong b i s u del ins
    • 3.行内块级元素 display:inline-block
      • 1.一行可以显示多个
      • 2.可以设置有效宽高
      • 常见的行内块级元素：input img

      • border-radius: 左上角 右上角 右下角 左下角;
    • padding 内边距
    • margin 外边距
    • content 因素内容
      • width
      • height
    • content-box：padding和border不被包含在定义的width和height之内。盒子的实际宽度=设置的width+padding+border
    • padding和border被包含在定义的width和height之内。
      盒子的实际宽度=设置的width（padding和border不会影响实际宽度）
    • inherit , 规定应从父元素继承 box-sizing 属性的值

github+hexo搭建博客

  1. 安装 nodejs

  2. 安装 git

  3. 安装Hexo

    • npm i -g hexo-cli

  4. 初始化一个新的博客

    • hexo init

  5. 安装依赖

    • npm i

  6. 开启本地服务器

    • hexo s
    • hexo s -p 5000

  7. 输入 地址查看博客

    • http://127.0.0.1:4000

  8. 写博客

    • 把写好的 markdown 文件，存放到 /source/_posts 文件夹 中即可
    • tips
      • 注意 文件名字不能过长

  9. 配置 NexT 主题（可选）

    • NexT 主题官方文档： [http://theme-next.iissnan.com]

  10. 完成部署GitHub

    • 每个 Github 账户都可以创建一个 YourName.github.io 的仓库，

    • 这个仓库的 master 分支是可以直接使用 YourName.github.io 在网络中访问的

    • 我们可以利用 Github 的这个功能来进行静态页面的展示。

    • 有两种部署的方式，一种是手动部署，一种是使用 hexo 命令行部署

      • 手动部署

        1. 清空之前生成的博客文件
          • hexo clean
          • 如果服务器开启中，可以使用 Ctrl + c 停止
        2. 生成博客文件
          • hexo g
        3. 进入 public 文件夹中，开启 bash 黑窗
          • 输入git init 初始化仓库，并使用 add 和 commit 提交
        4. 使用 git push 命令推送到 Github 仓库
          • git push https://github.com/YourName/YourName.github.io.git master --force
          • 注意后面需要加上 --force 强制覆盖
        5. 在浏览器输入 YourName.github.io 来访问你的博客

      • 使用 hexo 命令部署

        • 此方法需要配置 SSH 密钥来免密提交

        1. 安装 hexo 部署工具

          • npm i hexo-deployer-git --save

        2. 找到 ‘~/blog/_config.yml’ 文件，修改最后的代码：

        3. 清空之前生成的博客文件

          • hexo clean

        4. 生成博客文件

          • hexo g

        5. 部署到 Github

          • hexo d
          • 最后显示 INFO Deploy done: git 代表部署成功

        6. 在浏览器输入 YourName.github.io 来访问你的博客

  11. 总结常用命令

    • hexo clean
      • 清空 (hexo generate) 生成的博客
    • hexo g (hexo generate)
      • 生成博客文件
    • hexo s (hexo server)
      • 启动本地服务器
    • hexo d (hexo deploy)
      • 按照配置好的_config.yml文件，完成部署
    • hexo new “我的新帖子”
      • 创建一个新帖子

html

  1. 什么是html

    • 超文本标记语言 (Hyper Text Markup Language)
    • 是一种标记语言 (markup language)
    • 标记语言是一套标记标签 (markup tag)
    • HTML 使用标记标签来描述网页
    • HTML5中的文档类型
      • html:5
      • html:xt xhtml 1.0
      • html:4s html 4.01
    • 字符设定
      • <meta http-equiv="charset" content="utf-8">：HTML与XHTML中建议这样去写
      • <meta charset="utf-8">：HTML5的标签中建议这样去写
    • 网页的组成
      • 主要由 (文字, 图像, 超链接, 音频, 视频) 组成
    • 网页的形成
      1. 程序员编码
      2. 浏览器渲染
      3. 网页
    • 打开一个网页的过程中发生了什么

  2. html文档结构

  3. 标签语义化

    • 主要目的就是让大家直观的认识标签(markup)和属性(attribute)的用途和作用
    • 使用有意义的单词来作为标签的名字
    • 核心：在合适的地方使用合适的标签
    • 优点
      • 语义化HTML会使HTML结构变的清晰，有利于维护代码和添加样式
      • 让浏览器或是网络爬虫可以很好地解析，从而更好分析其中的内容
      • 便于团队开发和维护，语义化更具可读性，是下一步吧网页的重要动向，遵循W3C标准的团队都遵循这个标准，可以减少差异化。
    • 语义是否良好?
      • 当我们去掉CSS之后，网页结构依然组织有序，并且有良好的可读性
