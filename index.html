http状态码

2019-09-14
  • http状态码

  • 2开头 （请求成功）表示成功处理了请求的状态代码。

    • 200 （成功） 服务器已成功处理了请求。 通常，这表示服务器提供了请求的网页。
    • 201 （已创建） 请求成功并且服务器创建了新的资源。
    • 202 （已接受） 服务器已接受请求，但尚未处理。
    • 203 （非授权信息） 服务器已成功处理了请求，但返回的信息可能来自另一来源。
    • 204 （无内容） 服务器成功处理了请求，但没有返回任何内容。
    • 205 （重置内容） 服务器成功处理了请求，但没有返回任何内容。
    • 206 （部分内容） 服务器成功处理了部分 GET 请求。

  • 3开头 （请求被重定向）表示要完成请求，需要进一步操作。 通常，这些状态代码用来重定向。

    • 300 （多种选择） 针对请求，服务器可执行多种操作。 服务器可根据请求者 (user agent) 选择一项操作，或提供操作列表供请求者选择。
    • 301 （永久移动） 请求的网页已永久移动到新位置。 服务器返回此响应（对 GET 或 HEAD 请求的响应）时，会自动将请求者转到新位置。
    • 302 （临时移动） 服务器目前从不同位置的网页响应请求，但请求者应继续使用原有位置来进行以后的请求。
    • 303 （查看其他位置） 请求者应当对不同的位置使用单独的 GET 请求来检索响应时，服务器返回此代码。
    • 304 （未修改） 自从上次请求后，请求的网页未修改过。 服务器返回此响应时，不会返回网页内容。
    • 305 （使用代理） 请求者只能使用代理访问请求的网页。 如果服务器返回此响应，还表示请求者应使用代理。
    • 307 （临时重定向） 服务器目前从不同位置的网页响应请求，但请求者应继续使用原有位置来进行以后的请求。

  • 4开头 （请求错误）这些状态代码表示请求可能出错，妨碍了服务器的处理。

    • 400 （错误请求） 服务器不理解请求的语法。
    • 401 （未授权） 请求要求身份验证。 对于需要登录的网页，服务器可能返回此响应。
    • 403 （禁止） 服务器拒绝请求。
    • 404 （未找到） 服务器找不到请求的网页。
    • 405 （方法禁用） 禁用请求中指定的方法。
    • 406 （不接受） 无法使用请求的内容特性响应请求的网页。
    • 407 （需要代理授权） 此状态代码与 401（未授权）类似，但指定请求者应当授权使用代理。
    • 408 （请求超时） 服务器等候请求时发生超时。
    • 409 （冲突） 服务器在完成请求时发生冲突。 服务器必须在响应中包含有关冲突的信息。
    • 410 （已删除） 如果请求的资源已永久删除，服务器就会返回此响应。
    • 411 （需要有效长度） 服务器不接受不含有效内容长度标头字段的请求。
    • 412 （未满足前提条件） 服务器未满足请求者在请求中设置的其中一个前提条件。
    • 413 （请求实体过大） 服务器无法处理请求，因为请求实体过大，超出服务器的处理能力。
    • 414 （请求的 URI 过长） 请求的 URI（通常为网址）过长，服务器无法处理。
    • 415 （不支持的媒体类型） 请求的格式不受请求页面的支持。
    • 416 （请求范围不符合要求） 如果页面无法提供请求的范围，则服务器会返回此状态代码。
    • 417 （未满足期望值） 服务器未满足”期望”请求标头字段的要求。

  • 5开头（服务器错误）这些状态代码表示服务器在尝试处理请求时发生内部错误。 这些错误可能是服务器本身的错误，而不是请求出错。

    • 500 （服务器内部错误） 服务器遇到错误，无法完成请求。
    • 501 （尚未实施） 服务器不具备完成请求的功能。 例如，服务器无法识别请求方法时可能会返回此代码。
    • 502 （错误网关） 服务器作为网关或代理，从上游服务器收到无效响应。
    • 503 （服务不可用） 服务器目前无法使用（由于超载或停机维护）。 通常，这只是暂时状态。
    • 504 （网关超时） 服务器作为网关或代理，但是没有及时从上游服务器收到请求。
    • 505 （HTTP 版本不受支持） 服务器不支持请求中所用的 HTTP 协议版本。

跨域

2019-09-14
  • 跨域
  1. 什么是跨域？
    • Javascript处于安全方面的考虑（同源策略限制），不允许跨域调用其他页面的对象。
  2. 跨域的产生
    • Domains,protocols,posts 不一样就是跨域
      • Domains 域名
      • protocols 协议 （分为http和https）
  3. tips
    • 如果是protocols和ports不一致造成的跨域，前台没有办法解决
    • 域只通过URL的首部，也就是protocols+domain+port来判断是否跨域，不会去判断domain是否对应着相同的IP地址
  4. 解决跨域的方法
    • 前端
      • 使用HTML5新属性postMessage
      • 利用iframe和location.hash
      • 动态创建script标签
        • 虽然浏览器默认阻止跨域请求，但是不阻止在页面中引用其他域的文件。
        • 因此可以动态添加script标签。JSONP就是利用这一原理来实现ajax跨域的
      • 通过修改docment.domain来实现跨域
        • 该方法必须是在同主域，不同子域的情况下才生效
          • 如：morningstar.com 和 test.morningstar.com
          • 只要把两个页面 的document.domain都指向主域就可以了，比如document.domain='morningstar.com'

  1. 配置环境

    • java
      • 下载 java jdk
        • https://www.oracle.com/technetwork/java/javase/downloads/index.html
      • 安装
      • 配置java环境变量
        • 新增变量 JAVA_HOME
        • jdk所在的地址为变量值（我这里是D:\tools\Java\jdk1.8.0_181）的变量
      • 查看java环境变量是否配置成功
        • cmd java -version
    • android sdk
      • 下载 android sdk
        -
      • 配置安卓环境变量
        • 新增变量 ANDROID_SDK_HOME
        • 变量值为刚刚下载sdk的路径（我这面的这个路径是D:\tools\android-sdk）
      • 查看android环境变量是否配置成功
        • cmd adb
    • node 环境
      • 下载
        • https://nodejs.org/zh-cn/

  2. 安装 cordova

    • OS X和Linux
      • sudo npm install -g cordova
    • Windows
      • npm install -g cordova
    • 查看版本
      • cordova -v

  3. 创建 cordova项目

    • cordova create CordovaProject io.cordova.hellocordova CordovaApp
      • CordovaProject 是项目的目录名称
      • io.cordova.hellocordova 是默认的反向域值。包名
      • CordovaApp 是您应用的标题, 如 (微信)

  4. 平台

    • 添加平台
      • cordova platform add android
      • cordova platform add ios
      • cordova platform add browser
      • cordova platform add windows
    • 删除平台
      • cordova platform rm ios
      • cordova platform rm android
      • cordova platform rm browser
      • cordova platform rm windows
    • 检查当前的平台集
      • cordova platform ls
    • 检查是否满足构建平台的要求
      • cordova requirements

  5. 构建和运行(打包app)

    • 安卓
      • cordova build android 测试包
      • cordova build android --release 正式包
        • 出现 BUILD SUCCESSFUL 说明构建成功
      • cordova run android
        • 运行在外部模拟器或真实设备
    • ios
      • 使用 xcode 打包
    • tips
      • 项目不能位于中文路径下

  6. 插件

    • 查看当前安装的插件

      • cordova plugin ls

    • 腾讯X5 WebVie

      • cordova plugin add cordova-plugin-x5-webview

    • 地理位置

      • cordova plugin add cordova-plugin-geolocation

    • 全球化

      • cordova plugin add cordova-plugin-globalization

    • InAppBrowser

      • cordova plugin add cordova-plugin-inappbrowser
        • 此插件用于在Cordova应用程序中打开Web浏览器。

    • 启动页面和图标的设置

      • cordova plugin add cordova-plugin-splashscreen
      • config.xml配置

    • 热更新

      1. tips

        • cordova项目热更新分为两个部分，网页内容（web content)更新 和 Cordova插件(native side)更新。
        • web content 的运行是基于 native side。如:登录页面的微信登录，需要在项目中集成微信的插件才能实现登录。

      2. web代码的更新

        • 第一步: 添加插件 cordova plugin add cordova-hot-code-push-plugin

        • 第二步: 添加热更新工具 npm install -g cordova-hot-code-push-cli

        • 第三步: 添加cordova-hcp.json文件 cordova-hcp init

          • Project name: 你现在的项目名称所需.
          • Amazon S3 Bucket name:亚马逊上的S3桶的名字。可以跳过。
          • Amazon S3 region: 亚马逊S3地区。可以跳过。
          • iOS app identifier: 应用程序商店的应用程序ID。App升级是跳转应用商店进行升级。
          • Android app identifier: 可以App在应用商城的地址，或者是apk的下载地址。
          • Update method: 何时执行更新。
            • start: 在启动应用程序时安装更新。
            • resume: 在恢复应用程序(从背景移动到前景状态)或启动时，安装更新；默认使用。
            • now: 从服务器加载更新后立即安装。

        • 第四步: 在www文件下生成chcp.json 和 chcp.manifest文件 cordova-hcp build

          • 执行此命令会在chcp.json文件中增加”release”:”当前时间”字段。（默认使用时间戳，格式为：yyyy.MM.dd-HH.mm.ss），插件将版本号进行字符串相等比较来判断是否存在新版本。
          • “release”: “2017.06.07-16.30.20”,//唯一web项目版本号,用与热更新web内容的更新。（必需）
          • chcp.json: 包含发布相关信息：热更新代码版本号，应用 native side 版本号等等
          • chcp.manifest: 包含项目热更新代码(静态)文件信息：文件名和文件哈希值

  7. cordova + vue

    • 为了方便，不需要每次编译都拷贝文件，可直接在cordova项目根目录中创建vue项目。

      • 全局安装Vue-cli脚手架。

      • npm install -g vue

      • npm install -g vue-cli

      • 初始化一个vue项目 vue init webpack MyApp

      • 修改Vue项目config/index.js文件

        • 把build 输出路径修改为 cordova项目的 www 文件夹
        • npm run build 查看效果

      • 打包app 之前 检查 cordova 的配置是否完善 cordova requirements

      • 如配置完善,那么就可以用cordova命令打包app了

css笔记

2019-09-08
  • css

  1. 介绍

    • CSS (Cascading Style Sheets) 用于设置HTML页面中的文本内容（字体、大小、对齐方式等）、图片的外形（宽高、边框样式、边距等）以及版面的布局等外观显示样式
    • 名字
      • CSS样式表
      • 层叠样式表
      • 级联样式表
    • css 3大特性
      • 1.层叠性
        • 对于同一个元素的同一个属性，后面的样式会层叠(覆盖)前面的样式长江后浪推前浪，前浪死在沙滩上
      • 2.继承性
        • 继承父元素的文字大小,颜色
      • 3.优先级
        • 继承 < 通配符选择器 < 标签选择器（1） < 类选择器（10） < id选择器（100） < 行内样式（1000） < !important（无穷大）

  2. 引入方式

    • 外链样式

    • 内嵌样式

    • 行内样式

  3. 选择器

    • 通配符选择器

      • *
      • 匹配页面中所有的元素

    • id选择器

      • #
      • 元素的id值是唯一的 (相当于身份证)

    • 类选择器

      • .
      • 相当于人的名字

    • 属性选择器

      • input[type="password"]

    • 后代选择器

      • .red .blue

    • 子代选择器

      • .red>.blue

    • 并集选择器

      • .red , .blue

    • 交集选择器

      • p.red
      • 交集选择器中如果有标签选择器，标签选择器一定要写在最前面

    • 目标伪类选择器

      • :target
      • 改变目标的样式

    • 伪类选择器

      • :link 未访问的链接
      • :visited 已访问的链接
      • :hover 鼠标移动到链接上
      • :active 选定的链接
      • :link :visited只对a标签起作用

    • 伪元素选择器

      • ::first-letter
        • 文本的第一个单词或字
      • ::first-line
        • 文本第一行
      • ::selection
        • 可改变选中文本的样式；
      • ::before
      • ::after
      • ::before和::after一定要配合content一起使用

    • 结构(位置)伪类选择器

      • :nth-child(n)
        • 匹配属于其父元素的第 N 个子元素，不论元素的类型
      • :first-child
        • 选取属于其父元素的首个子元素的指定选择器
      • :last-child
        • 选取属于其父元素的最后一个子元素的指定选择器
      • :nth-last-child(n)
        • 选择器匹配属于其元素的第 N 个子元素的每个元素，不论元素的类型，从最后一个子元素开始计数。
      • n 可以是数字、关键词或公式

  4. 文字样式

    • 字体大小

      • font-size
        • 现在网页中普遍使用14px+
        • 尽量使用偶数的数字字号，ie6等老式浏览器支持奇数会有bug

    • 字体粗细

      • font-weight
        • normal
        • bold
        • bolder
        • lighter
        • 100~900（100的整数倍）

    • 字体样式/字体风格

      • font-style
        • normal
        • italic 斜体
        • oblique 倾斜的字体样式

    • 字体系列

      • font-family
        • Microsoft YaHei 微软雅黑
        • Sim Sun 宋体
        • 尽量使用系统默认字体，保证在任何用户的浏览器中都能正确显示

    • 文本颜色

      • color

    • 文本缩进

      • text-indent
        • em
        • px
        • %
        • 负数

    • 文本对齐

      • text-align
        • center
        • right
        • left

    • 文本修饰线

      • text-decoration
        • underline 定义文本下的一条线。下划线 也是我们链接自带的
        • overline 定义文本上的一条线。
        • line-through 定义穿过文本下的一条线

    • 文字间隔

      • word-spacing 单词间距
        • 对于中文无效
        • 设置单词之间的间距
      • letter-spacing 字间距
        • 中文设置的每个字之间的间距
        • 英文单词设置每个字母之间的间距

    • 文字阴影

      • text-shadow:水平位置 垂直位置 模糊距离 阴影颜色;
        • text-shadow:20px 50px 5px rgba(255,0,0,.1);
        • 设置文字阴影的时候，水平位置和垂直位置一定不能少，其他可以省略
      • 文字多阴影
        • text-shadow:2px 2px 5px blue,5px 5px 5px red;

    • 换行

      • white-space
        • normal
          • 使用浏览器默认的
        • nowrap
          • 文本不会换行，文本会在在同一行上继续，直到遇到
            标签为止。
        • wrap
          • 换行

    • 自动换行

      • word-break
        • normal
          • 使用浏览器默认的换行规则。
        • break-all
          • 允许在单词内换行。
        • keep-all
          • 只能在半角空格或连字符处换行。

    • 文字溢出

      • text-overflow

        • clip
          • 不显示省略标记（…），而是简单的裁切
        • ellipsis
          • 当对象内文本溢出时显示省略标记（…）

      • 如果需要实现一个文本超出盒子加上省略号的效果，我们使用text-overflow，但是一定要配合:white-space:nowrap和overflow:hidden一起使用才能实现效果

    • 复合属性

      • font:font-style font-weight font-size font-family;
        • 1.font-size 和 font-family的顺序不能打乱而且位置不能改变
        • 2.font-size 和 font-family一定不能少，其他的可以选

  5. 单位

    • 相对单位

      • em

        • 基准点为 父节点 字体的大小
        • 如果自身定义了font-size按自身来计算（浏览器默认字体是16px)

      • rem

        • 相对 根节点html 的字体大小来计算

      • vh 视窗高度

        • 1vh 等于视窗高度的1%

      • vw 视窗宽度

        • 1vw 等于视窗宽度的1%

      • vmin

        • vw 和 vh 中较小的那个。

      • vmax

  8. 定位

    • position

      1. absolute (绝对定位)

        • 生成绝对定位的元素，相对于 static 定位以外的第一个父元素进行定位。
          • 若所有父元素都没有定位，以浏览器为准对齐(document文档)。
        • 可通过z-index进行层次分级。
        • 脱标，不占原来的位置
        • 可以改变元素的显示方式
        • 子绝父相 （子元素是绝对定位，父元素要用相对定位）
        • 如果只设置了定位方式，而没有设置偏移量，那么元素的位置不变

      2. relative (相对定位)

        • 不脱标，占原来的位置
        • 相对与自身原来在标准流中的位置来做位置移动
        • 可通过z-index进行层次分级 。
        • 如果只设置了定位方式，而没有设置偏移量，那么元素的位置不变
3. `fixed` (固定定位)
    - 生成绝对定位的元素，相对于浏览器窗口进行定位。
    - 可通过`z-index`进行层次分级
    - tips
        - ie6等低版本浏览器不支持固定定位。
4. `static` (静态定位)
    - 默认定位，元素出现在正常的流中（`忽略 top, bottom, left, right 或者 z-index声明`)

5. `inherit`
    - 规定应该从父元素继承 position 属性的值
6. `sticky` (粘性定位)
    - 可以说是相对定位relative和固定定位fixed的结合；它主要用在对scroll事件的监听上；简单来说，在滑动过程中，某个元素距离其父元素的距离达到sticky粘性定位的要求时(比如top：100px)；position:sticky这时的效果相当于fixed定位，固定到适当位置
    - tips
        - sticky不会触发BFC
        - `z-index无效`
        - 当父元素的height：100%时，页面滑动到一定高度之后sticky属性会失效。
        - 父元素不能有overflow:hidden或者overflow:auto属性。
        - 父元素高度不能低于sticky高度，必须指定top、bottom、left、right4个值之一

  1. 层级

    • z-index
      • 定位的元素才有 z-index 值
      • 如果 z-index 相同, 后面的元素会压住前面的元素
      • z-index值没有单位，就是一个正整数。默认的z-index值是0
      • 从父现象：父亲怂了，儿子再牛逼也没用。意思是，如果父亲1比父亲2大，那么，即使儿子1比儿子2小，儿子1也能在最上层。

      • iframe等于打开一个新的网页，所有的JS/CSS全部加载一遍，内存会*2，无法释放，典型的内存泄露(以EXTJS3,+ FF为例，每次刷新页面，内存增加8M，不释放！如果2个小时不关，浏览器内存消耗500M)
      • iframe会阻塞主页面的Onload事件；
      • iframe和主页面共享连接池，而浏览器对相同域的连接有限制，所以会影响页面的并行加载。
      • 移动端对iframe不友好frame是(几乎)不能访问外部数据的，js的作用域也会很奇怪。
      • 如果有局部刷新需求，重型的上MVVM或者databinding框架，轻型的上ajax。

  9. HTML5全局属性

    • data-* 使用 data-* 属性来嵌入自定义数据。
      • 属性名不应该包含任何大写字母，并且在前缀 “data-“ 之后必须有至少一个字符,属性值可以是任意字符串。
    • hidden
      • 它规定元素仍未或不再相关。浏览器不应显示已规定 hidden 属性的元素。hidden 属性也可用于防止用户查看元素，直到匹配某些条件（比如选择了某个复选框）。然后，JavaScript 可以删除 hidden 属性，以使此元素可见。
    • spellcheck
      • 规定是否对元素进行拼写和语法检查。
    • tabindex
      • abindex 属性规定元素的 tab 键控制次序（当 tab 键用于导航时）
      • 以下元素支持 tabindex 属性：<a>, <area>, <button>, <input>, <object>, <select> 以及 <textarea>。
    • contenteditable
      • 规定元素内容是否可编辑。
      • 可继承

  10. seo优化

    • 标题 title
    • 关键词 keywords
    • 描述 description
    • 图片 alt
    • 图片大小
    • h1 标签
    • a 标签
    • 注意
      • 尽可能少的使用无语义的标签div和span；
      • 在语义不明显时，既可以使用div或者p时，尽量用p, 因为p在默认情况下有上下间距，对兼容特殊终端有利；
      • 不要使用纯样式标签，如：b、font、u等，改用css设置。
      • 需要强调的文本，可以包含在strong或者em标签中（浏览器预设样式，能用CSS指定就不用他们），strong默认样式是加粗（不要用b），em是斜体（不用i）；
      • 使用表格时，标题要用caption，表头用thead，主体部分用tbody包围，尾部用tfoot包围。表头和一般单元格要区分开，表头用th，单元格用td；
      • 表单域要用fieldset标签包起来，并用legend标签说明表单的用途；
      • 每个input标签对应的说明文本都需要使用label标签，并且通过为input设置id属性，在lable标签中设置for=someld来让说明文本和相对应的input关联起来。

      • 废除xmp使用code替代;
      • 废除nextid使用guids;
      • 废除plaintex使用“text/plian”（无格式正文）MIME类型替代。
