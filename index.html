    • 介于Java与HTML之间、基于对象和事件驱动的编程语言
    • 可直接运行在Web浏览器中的弱类型脚本语言
    • JavaScript 对大小写是敏感的。

      • 全局变量会在页面关闭后被删除。
      • 局部变量会在函数运行以后被删除。

  3. 数据类型

    1. number （数字）

      • Infinity (或 -Infinity) 是 JavaScript 在计算数时超出最大可能数范围时返回的值

      • 方法

        1. isNaN() 确定某个值是否是数

        2. toString() 以字符串返回数值

        3. toExponential(amount) 返回字符串值，它包含已被四舍五入并使用指数计数法的数字

        4. toFixed(amount) 返回字符串值，它包含了指定位数小数的数字

        5. toPrecision() 返回字符串值，它包含了指定长度的数字

        6. valueOf() 以数值返回数值

        7. Number() 返回数字，由其参数转换而来

        8. parseFloat() 解析其参数并返回浮点数。

        9. parseInt() 解析其参数并返回整数。

    2. string (字符串)

      • 单引号(‘’) 或者 双引号(“”) 再或者 反单引号(``) 内的皆为字符串
      • 操作字符串

      1. string.length

        • 获取字符串的长度

      2. string.indexOf('name')

        • 返回字符串中指定文本首次出现的索引,无法设置更强大的搜索值（正则表达式）
        • 如果没找到对应的字符函数返回-1

      3. string.lastIndexOf('name')

        • 返回指定文本在字符串中最后一次出现的索引

      4. string.search('name')

        • 搜索特定值的字符串，并返回匹配的位置,无法设置第二个开始位置参数

      5. string.slice(start, end)

      6. string.substring(start, end)

      7. string.substr(start, length)

      8. string.replace('x','y')

        • 替换 不会改变调用它的字符串。它返回的是新字符串。

      9. string.toLowerCase()

        • 把字符串转换为大写

      10. string.toLowerCase()

        • 把字符串转换为小写

      11. string.concat('','hello')

        • 连接两个或多个字符串

      12. string.trim()

        • 删除字符串两端的空白符

      13. string.charAt()

        • 返回字符串中指定下标（位置）的字符串

      14. string.charCodeAt()

        • 返回字符串中指定索引的字符 unicode 编码

      15. string.split(',')

        • 将字符串转换为数组

      16. fromCharCode()

        • 将 Unicode 转换为字符串

      17. localeCompare

        • 用本地特定的顺序来比较两个字符串

      18. match()

        • 找到一个或多个正则表达式的匹配

    3. Array （数组）

      • 数组用于在单一变量中存储多个值
      • 语法

      1. var arr = []

      2. var arr1 = [1,2,3,4]

      3. var arr2 = new Array()

      • 操作数组

        1. 遍历数组 for循环

        2. 遍历数组 Array.foreach(function)

        3. push() 在数组结尾处）向数组添加一个新的元素

        4. arr[arr.length]= ‘gg’

        5. Array.isArray()

        6. join() 方法也可将所有数组元素结合为一个字符串

        7. pop() 方法从数组中删除最后一个元素

        8. shift() 方法会删除首个数组元素，并把所有其他元素“位移”到更低的索引

        9. unshift() 方法（在开头）向数组添加新元素，并“反向位移”旧元素

        10. delete 会在数组留下未定义的空洞。请使用 pop() 或 shift() 取而代之

        11. splice() 方法可用于向数组添加新项

        12. concat() 方法通过合并（连接）现有数组来创建一个新数组

        13. slice() 方法创建新数组。它不会从源数组中删除任何元素

        14. sort() 方法以字母顺序对数组进行排序

        15. reverse() 方法反转数组中的元素

        16. map() 方法通过对每个数组元素执行函数来创建新数组,不会对没有值的数组元素执行函数,不会更改原始数组

  6. 作用域 及 变量提升 及闭包

    • 作用域：

      1. js中的作用域，也叫词法作用域

      2. js中只有函数能够形成一个作用域，所以，又叫函数作用域

      3. 变量的作用范围在书写代码的时候，就定下来了，与代码怎么调用无关

      4. 对于函数来说，创建好了函数以后，函数的作用域就定下来了，也就是作用域链也就定下来了

      5. 也就是说不管在哪个位置调用的函数，都是会回到创建它的环境中去执行！

    • 闭包

      1. 函数内部声明的变量，在函数外部无法访问，这个时候就形成了一个封闭的空间，也就是闭包

      2. 闭包包括了当前函数的函数体和函数所处的环境（作用域）

      3. 闭包其实也就是对作用域的应用

      4. 闭包的模型，也就是函数内部嵌套函数

      5. 闭包的好处：可以包含函数内部私有状态，也就是保护函数内部变量

    • JavaScript 应用程序的模块打包器(module bundler)
    • 分析你的项目结构，找到JavaScript模块以及其它的一些浏览器不能直接运行的拓展语言（Sass，TypeScript等），并将其转换和打包为合适的格式供浏览器使用。
    • SPA（single page web application） 单页应用

  • 重点内容

    • 入口(entry)
    • 输出(output)
    • 加载器(loader)
    • 插件(plugins)

  • 安装

    • npm install webpack -g

  • 配置

    • webpack.config.js

    • webpack-dev-server

      • 自动编译刷新浏览器
      • 安装
        • npm init -y
          • 生成package.json项目描述文件
        • npm install webpack-dev-server -g
      • 运行
        • webpack-dev-server --inline

    • 打包less资源

      • npm install less less-loader style-loader css-loader --save-dev

    • 打包sass 资源

      • npm install node-sass sass-loader css-loader style-loader --save-dev

    • 打包url()请求的资源

      • npm install url-loader file-loader --save-dev

    • es6 转 es5

      • npm install babel-core babel-loader babel-preset-es2015 babel-plugin-transform-runtime --save-dev

    • html-webpack-plugin

      • npm i html-webpack-plugin --save-dev

    • html-loader

      • npm i -D html-loader

  • 运行

    • npm run dev

服务器/客户端

  1. 什么是服务器

    • 服务器就是通过 安装服务器软件，向外提供服务的计算机。
    • 根据服务器软件的 功能，名称，安装的系统这些不同的标准，可以划分出不同类型的服务器。
    • HTTP服务器主要提供网站浏览服务，可以结合某编程语言进行开发，我们称之为服务器开发。
    • 服务端的编程语言
      • php、java、.net、Python、Ruby、Perl

  2. 常见的服务器软件有：

    • 文件服务器：
      • Server-U、FileZilla、VsFTP 等 (FTP是File Transfer Protocol文件传输协议)；
    • 数据库服务器：
      • oracle、mysql、SQL server、DB2、ACCESS等；
    • 邮件服务器：
      • Postfix、Sendmail等；
    • HTTP服务器：
      • Apache、Nginx、IIS、Tomcat、NodeJS等；

  3. 服务器类型

    • 按软件功能类型划分：
      • 文件服务器、数据库服务器、邮件服务器、Web服务器等；
    • 按安装操作系统划分：
      • Linux服务器、Windows服务器等；
    • 按应用软件名称划分：
      • Apache服务器、Nginx 服务器、IIS服务器、Tomcat服务器、 mysql服务器、oracle服务器、 Node服务器等；

  4. 什么是客户端

    • 客户端：
      • 安装客户端软件，索取服务一方（向服务器获取）
      • 电脑，手机，iPad平板安装了客户端软件，可以称之为客户端。
    • 服务端：
      • 安装服务端软件，提供服务一方（向客户端提供）

  5. 客户端分类

    • 一. CS软件（client/server） 客户端+服务器 的应用
      • 手机QQ，LOL游戏 , 手机微信
        1. 需要安装，手机、电脑桌面应用
        2. 性能更高效，更稳定，使用更流畅
        3. 开发及维护难度增大
        4. 兼容各种系统平台
        • 二. bs软件（browser/server） 浏览器+服务器 的应用( 一种特殊的CS 软件)
      • 网页QQ，新浪网站，网页地图
        1. 不用安装，运行在浏览器网页应用
        2. 性能相对较低，流畅性弱（依赖浏览器）
        3. 开发维护相对更快
        4. 兼容浏览器

  6. 网络基础 IP , 域名 , 端口, DNS服务.

    • ip
      • IP地址 每个连接在互联网上的电脑主机在网络上的唯一标识（身份证，手机电话号码）
      • 形式: 一个32位地址(192.168.10.100)
      • 查看本机IP地址: ifconfig
      • 查看网站IP地址: ping www.baidu.com
    • 域名
      • 由于IP地址基于数字，不方便记忆，于是便用域名来代替IP地址，域名是一个IP地址的“面具”
      • 形式: www.baidu.com
      • 查看域名对应的IP地址: ping www.baidu.com
    • DNS服务
      • DNS（Domain Name System）网上作为域名和IP地址相互映射的一个分布式数据库， 它能够使用户更方便的通过域名形式访问网站。
      • 就是指记录IP地址和域名之间对应关系的服务器电脑。
      • DNS寻址:先找本机hosts文件、再找DNS服务器( 专干对应域名IP的活 ) 刷新DNS： ipconfig /flushdns
    • 端口
      • 端口号是计算机与外界通讯交流的出口，每个端口对应不同的服务。
      • 查看端口占用情况： netstat -an 常见端口号 80、8080、3306、21、22
    • url
      • http：// 192.168.199.117:8080/index.html
      • 协议名+ 服务器ip + 端口号 + 请求文件名称

  1. vue介绍

      • webpack
        • 基于webpack的开发环境，集成了代码检查、测试、热部署等功能
      • webpack-simple
        • 基于webpack的简单版开发环境
      • browerify
        • 基于browerify的开发环境，集成了代码检查、测试、热部署等功能
      • browerify-simple
        • 基于browerify的简单版开发环境
      • simple
        • 只提供解析.vue文件的普通开发环境
    • 项目结构
      • build
        • webpack相关配置与脚本
      • config
        • 配置文件，主要用于区分开发环境、测试环境、线上环境
      • src
        • 项目源码及引用的资源
      • static
        • 不需要webpack处理的静态资源
      • test
        • 测试脚本

  4. 安装依赖

    • npm i

  5. 运行项目

    • npm run dev

    • M
      • Model(数据模型)
    • V
      • View(视图模型,负责将数据模型转化成UI展现出来，就是那些DOM结构)
    • VM
      • ViewModel(一个同步View和Model的对象)

  8. 指令

    • 插值表达式

    • v-text

      • 将一个变量的值渲染到指定的元素中

    • v-html

      • 被插入的内容都会被当做 HTML

    • v-bind

      • 可以给html元素或者组件动态地绑定一个或多个特性，例如动态绑定style和class
      • 使用方式

    • v-for

    • v-model

      • 默认是string类型
        • 可以通过.number方式确保获取到的是数字类型
      • 在表单控件或者组件上创建双向绑定
      • 支持的元素
        • input
        • select
        • textarea
        • components（Vue中的组件）
      • <input type="text" v-model="uname" />

    • v-on

      • 绑定事件监听，

      • 表达式可以是一个方法的名字或一个内联语句，如果没有修饰符也可以省略，

      • 用在普通的html元素上时，只能监听原生DOM 事件

      • 常用事件

        • click
        • keydown
        • keyup
        • mousedown
        • mouseover
        • submit

      • 阻止默认行为

        • <form v-on:submit.prevent></form>

      • 阻止冒泡

        • .stop

      • 优先触发

        • .capture

      • .self

        • 只有自己能触发，子元素无法触发

      • .once

        • 只能提交一次

      • v-pre

        • 跳过元素的编译，显示原始标签

      • 阻止提交

        • .prevent
          • 只作用于 a 和 form

      • .lazy

        • 相当于监听change操作,失去焦点

      • 按键修饰符

        • <input type="text" @keydown.enter="kd1">
          • 当按下回车键时触发kd1事件
        • enter 回车
        • tab
        • delete 删除/退格
        • esc
        • space
        • up
        • down
        • left
        • right

    • v-if 和 v-show

      • v-if

        • 根据表达式的真假值,是否渲染
        • 在切换时元素及它的数据绑定被销毁并重建

      • v-show

        • 根据表达式的真假值，切换元素的 display CSS 属性

    • v-cloak

      • 实例结束编译后自动移除

    • ref

    • keep-alive

      • 如果把切换出去的组件保留在内存中，可以保留它的状态或避免重新渲染

    • 自定义指令

      • 利用Vue.directive('指令id',{inserted:function(el,binding){}})

flex布局

display:felx;

  1. 定义容器内元素排列方向 (默认横向排列)

    • flex-direction
      • row;
        • 水平左到右
      • row-reverse
        • 水平右到左
      • column
        • 垂直，上到下
      • column-reverse
        • 垂直，下到上

  2. 定义容器内元素换行 (默认不换行)

    • flex-wrap
      • nowrap
        • 元素不换行
      • wrap
        • 换行
      • wrap-reverse
        -

  3. 元素在主轴（页面）上的排列

    • justify-content
      • center
      • flex-start
        • 左对齐
      • flex-end
        • 右对齐
      • space-between
        • 两端对齐，项目之间间隔相等；
      • space-around
        • 每个元素两侧的间隔相等

  4. 元素垂直对齐

    • align-items

      • flex-start

      • flex-end

      • center

        • 居中

      • baseline

        • 如弹性盒子元素的行内轴与侧轴为同一条，则该值与’flex-start’等效。其它情况下，该值将参与基线对齐

  5. 元素靠边位置 （垂直）

    • align-content
      • flex-start
      • flex-end
      • stretch
        • 元素位于容器的中心
      • center
        • 默认值
        • 元素被拉伸以适应容器
      • space-between
      • space-around

  6. flex-flow属性

    • flex-direction和flex-wrap的简写
    • 默认row nowrap
