github+hexo搭建博客

  1. 安装 nodejs

  2. 安装 git

  3. 安装Hexo

    • npm i -g hexo-cli

  4. 初始化一个新的博客

    • hexo init

  5. 安装依赖

    • npm i

  6. 开启本地服务器

    • hexo s
    • hexo s -p 5000

  7. 输入 地址查看博客

    • http://127.0.0.1:4000

  8. 写博客

    • 把写好的 markdown 文件，存放到 /source/_posts 文件夹 中即可

  9. 配置 NexT 主题（可选）

    • NexT 主题官方文档： [http://theme-next.iissnan.com]

  10. 完成部署GitHub

    • 每个 Github 账户都可以创建一个 YourName.github.io 的仓库，

    • 这个仓库的 master 分支是可以直接使用 YourName.github.io 在网络中访问的

    • 我们可以利用 Github 的这个功能来进行静态页面的展示。

    • 有两种部署的方式，一种是手动部署，一种是使用 hexo 命令行部署

      • 手动部署

        1. 清空之前生成的博客文件
          • hexo clean
          • 如果服务器开启中，可以使用 Ctrl + c 停止
        2. 生成博客文件
          • hexo g
        3. 进入 public 文件夹中，开启 bash 黑窗
          • 输入git init 初始化仓库，并使用 add 和 commit 提交
        4. 使用 git push 命令推送到 Github 仓库
          • git push https://github.com/YourName/YourName.github.io.git master --force
          • 注意后面需要加上 --force 强制覆盖
        5. 在浏览器输入 YourName.github.io 来访问你的博客

      • 使用 hexo 命令部署

        • 此方法需要配置 SSH 密钥来免密提交

        1. 安装 hexo 部署工具

          • npm i hexo-deployer-git --save

        2. 找到 ‘~/blog/_config.yml’ 文件，修改最后的代码：

        3. 清空之前生成的博客文件

          • hexo clean

        4. 生成博客文件

          • hexo g

        5. 部署到 Github

          • hexo d
          • 最后显示 INFO Deploy done: git 代表部署成功

        6. 在浏览器输入 YourName.github.io 来访问你的博客

  11. 总结常用命令

    • hexo clean
      • 清空 (hexo generate) 生成的博客
    • hexo g (hexo generate)
      • 生成博客文件
    • hexo s (hexo server)
      • 启动本地服务器
    • hexo d (hexo deploy)
      • 按照配置好的_config.yml文件，完成部署
    • hexo new “我的新帖子”
      • 创建一个新帖子

HTML笔记

  1. 什么是html

    • 超文本标记语言 (Hyper Text Markup Language)
    • 是一种标记语言 (markup language)
    • 标记语言是一套标记标签 (markup tag)
    • HTML 使用标记标签来描述网页
    • HTML5中的文档类型
      • html:5
      • html:xt xhtml 1.0
      • html:4s html 4.01
    • 字符设定
      • <meta http-equiv="charset" content="utf-8">：HTML与XHTML中建议这样去写
      • <meta charset="utf-8">：HTML5的标签中建议这样去写
    • 网页的组成
      • 主要由 (文字, 图像, 超链接, 音频, 视频) 组成
    • 网页的形成
      1. 程序员编码
      2. 浏览器渲染
      3. 网页
    • 打开一个网页的过程中发生了什么

  2. html文档结构

  3. 标签语义化

    • 主要目的就是让大家直观的认识标签(markup)和属性(attribute)的用途和作用
    • 使用有意义的单词来作为标签的名字
    • 核心：在合适的地方使用合适的标签
    • 优点
      • 语义化HTML会使HTML结构变的清晰，有利于维护代码和添加样式
      • 让浏览器或是网络爬虫可以很好地解析，从而更好分析其中的内容
      • 便于团队开发和维护，语义化更具可读性，是下一步吧网页的重要动向，遵循W3C标准的团队都遵循这个标准，可以减少差异化。
    • 语义是否良好?
      • 当我们去掉CSS之后，网页结构依然组织有序，并且有良好的可读性

  9. HTML5全局属性

    • data-* 使用 data-* 属性来嵌入自定义数据。
      • 属性名不应该包含任何大写字母，并且在前缀 “data-“ 之后必须有至少一个字符,属性值可以是任意字符串。
    • hidden
      • 它规定元素仍未或不再相关。浏览器不应显示已规定 hidden 属性的元素。hidden 属性也可用于防止用户查看元素，直到匹配某些条件（比如选择了某个复选框）。然后，JavaScript 可以删除 hidden 属性，以使此元素可见。
    • spellcheck
      • 规定是否对元素进行拼写和语法检查。
    • tabindex
      • abindex 属性规定元素的 tab 键控制次序（当 tab 键用于导航时）
      • 以下元素支持 tabindex 属性：<a>, <area>, <button>, <input>, <object>, <select> 以及 <textarea>。
    • contenteditable
      • 规定元素内容是否可编辑。
      • 可继承
