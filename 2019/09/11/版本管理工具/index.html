Git

  1. git的介绍

    • 官网
    • Git是一款免费、开源的分布式 版本控制软件
    • Git的优势
      • Git的速度快，性能好
      • 采用分布式存储方式，不需要服务器就可以运行

  2. 安装git

    • 下载：https://git-scm.com/downloads

  3. 用户信息配置

    • 设置用户名
      • git config -- global user.name '你在github上注册的用户名'
    • 设置用户邮箱
      • git config -- global user.email '注册时候的邮箱'
    • 查看用户配置信息
      • git config --list

  4. git代码提交5步曲

    1. 提交代码第一步：git status 查看当前状态

    2. 提交代码第二步：git add . (不要忘记点) 或者 git add xxx

    3. 提交代码第三步：git commit -m “修复XXbug” 推送修改到本地git库中

    4. 提交代码第四步：git pull <远程主机名> <远程分支名> 取回远程主机某个分支的更新，再与本地的指定分支合并。
      例如：项目所在的远程主机是origin 远程主分支名为master 我们就可以写成 pull origin master 需要输入你的git账号和密码才能拉取。

    5. 提交代码第五步：git push <远程主机名> <远程分支名> 把当前提交到git本地仓库的代码推送到远程主机的某个远程分之上

  5. Git的4种文件状态

    1. 未追踪（untracked）

    2. 已提交（committed）

    3. 已修改（modified）

    4. 已暂存（staged）

  6. .gitignore 文件

    • 忽略一些不需要被git管理的文件
    • build/
      • 忽略 build/ 目录下的所有文件
    • 使用 # 表示注释
    • 使用 * 表示通配符

  7. 常用命令

    • 查看版本
      • git --version
    • 初始化仓库
      • git init
    • 跟踪文件
      • git add 文件路径
    • 提交更新
      • git commit -m "提交说明"
    • 查看文件状态
      • git status
      • git stauts -s
    • 查看提交日志
      • git log
    • 版本回退
      • git reset --hard [版本号]
        • 恢复到已经提交的某一个版本中
      • git reset --hard head~1
        • 表示回到上一次提交
    • 查看分支
      • git branch
        • 当前分支会标有一个*
    • 创建分支
      • git branch [分支名称]
        • 分支中的代码，与创建那一刻主分支中的内容完全相同
    • 切换分支
      • git checkout [分支名称]
    • 创建并切换分支
      • git checkout -b [分支名称]
    • 合并分支
      • git merge [分支名称]
        • 将其他分支合并到当前分支
    • 删除分支
      • git branch -d [分支名称]
    • 将代码提交到远程仓库
      • git push [仓库地址] master
    • 获取仓库中的最新内容
      • git pull [仓库地址] master
    • 克隆仓库到本地
      • git clone [仓库地址] [自定义本地仓库名]
    • 文件对比
      • git diff
        • 将工作区与暂存区或者仓库对比
      • git diff [版本号1] [版本号2] [对比的文件路径]
        • 对比仓库区两次提交的差异

  8. 合并分支冲突

    • 合并分支时出现冲突只能手动处理文件，然后，再次提交

  9. SSH介绍

    • ssh是一种安全的传输模式
    • 创建SSH Key
      • ssh-keygen -t rsa
    • 在文件路径 C:\用户\当前用户名\ 找到 .ssh 文件夹
    • 文件夹中有两个文件
      • 私钥：id_rsa
      • 公钥：id_rsa.pub
    • 在 github -> settings -> SSH and GPG keys页面中，新创建SSH key
    • 粘贴 公钥 id_rsa.pub 内容到对应文本框中
