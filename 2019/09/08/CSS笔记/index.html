css笔记

  1. 介绍

    • CSS (Cascading Style Sheets) 用于设置HTML页面中的文本内容（字体、大小、对齐方式等）、图片的外形（宽高、边框样式、边距等）以及版面的布局等外观显示样式
    • 名字
      • CSS样式表
      • 层叠样式表
      • 级联样式表
    • css 3大特性
      • 1.层叠性
        • 对于同一个元素的同一个属性，后面的样式会层叠(覆盖)前面的样式长江后浪推前浪，前浪死在沙滩上
      • 2.继承性
        • 继承父元素的文字大小,颜色
      • 3.优先级
        • 继承 < 通配符选择器 < 标签选择器（1） < 类选择器（10） < id选择器（100） < 行内样式（1000） < !important（无穷大）

      • W3C标准盒子/内容盒子

      • 盒子模型包含5个属性
        • padding
        • margin

    • 内边距

      • 边框与内容之间的距离

      • padding

        • padding-left

        • padding-right

        • padding-top

        • padding-bottom

        • padding: 10px

          • 表示上,下,左,右内边距都为10px

        • padding: 10px 15px

          • 表示上,下内边距为10px，左,右内边距为15px

        • padding: 10px 5px 2px

          • 表示上内边距为10px，左,右内边距为5px，下内边距为2px

        • padding: 1px 2px 3px 4px

          • 表示上,下,左,右内边距分别为1px ,2px ,3px, 4px

    • 外边距

      • margin

        • 设置外边距会在元素之间创建“空白”， 这段空白通常不能放置其他内容

        • 取值顺序跟内边距相同。

        • 外边距实现盒子居中

          • 必须是块级元素。
          • 盒子必须指定了宽度（width）
          • 左右的外边距都设置为auto

        • tips

          • margin 存在兼容问题，各浏览器默认显示或不一样

          • 行内元素是只有左右外边距的，是没有上下外边距的。 内边距，在ie6等低版本浏览器也会有问题。

          • 外边距塌陷

            • 相邻的两个块元素相遇时，上面盒子margin-bottom： 10px；下面盒子margin-top：5px；则两盒子相距10px
              • 两盒子之间的距离会取值较大者
            • 解决方案：
              1. 可以为父元素定义1像素的上边框或上内边距。
              2. 可以为父元素添加 overflow:hidden。

          • ie6下面margin 加倍

  7. 元素类型/显示方式

    • display

      • flex

        • 见笔记 flex布局.md

      • block

        • 元素显示方式为 块级
        • 独占一行
        • 宽度(width)、高度(height)、内边距(padding)和外边距(margin)都可控制;

      • inline-block

        • 元素显示方式为 行内块
        • 和相邻的行内元素 及 行内块元素 在同一行;
        • 宽度(width)、高度(height)、内边距(padding)和外边距(margin)都可控制;

      • inline

        • 元素显示方式为 行内
        • 和相邻的行内元素 及 行内块元素在同一行;
        • 宽度(width)、高度(height)、内边距的top/bottom(padding-top/padding-bottom)和外边距的top/bottom(margin-top/margin-bottom)都不可改变，宽高就是里面文字或图片的大小;

      • visibility:hidden 和 display: none

        • visibility:hidden 隐藏某个元素, 仍然占用原有的空间空间(即,元素虽然被隐藏了,但仍然会影响布局)

        • display:none 隐藏某个元素, 不占用原有的空间空间(即,元素被隐藏,不会影响布局)

      • 行内块级元素不能转换为行内元素，转换是无效的

  8. 定位

    • position

      1. absolute (绝对定位)

        • 生成绝对定位的元素，相对于 static 定位以外的第一个父元素进行定位。
          • 若所有父元素都没有定位，以浏览器为准对齐(document文档)。
        • 可通过z-index进行层次分级。
        • 脱标，不占原来的位置
        • 可以改变元素的显示方式
        • 子绝父相 （子元素是绝对定位，父元素要用相对定位）
        • 如果只设置了定位方式，而没有设置偏移量，那么元素的位置不变

      2. relative (相对定位)

        • 不脱标，占原来的位置
        • 相对与自身原来在标准流中的位置来做位置移动
        • 可通过z-index进行层次分级 。
        • 如果只设置了定位方式，而没有设置偏移量，那么元素的位置不变
3. `fixed` (固定定位)
    - 生成绝对定位的元素，相对于浏览器窗口进行定位。
    - 可通过`z-index`进行层次分级
    - tips
        - ie6等低版本浏览器不支持固定定位。
4. `static` (静态定位)
    - 默认定位，元素出现在正常的流中（`忽略 top, bottom, left, right 或者 z-index声明`)

5. `inherit`
    - 规定应该从父元素继承 position 属性的值
6. `sticky` (粘性定位)
    - 可以说是相对定位relative和固定定位fixed的结合；它主要用在对scroll事件的监听上；简单来说，在滑动过程中，某个元素距离其父元素的距离达到sticky粘性定位的要求时(比如top：100px)；position:sticky这时的效果相当于fixed定位，固定到适当位置
    - tips
        - sticky不会触发BFC
        - `z-index无效`
        - 当父元素的height：100%时，页面滑动到一定高度之后sticky属性会失效。
        - 父元素不能有overflow:hidden或者overflow:auto属性。
        - 父元素高度不能低于sticky高度，必须指定top、bottom、left、right4个值之一

  1. 层级

    • z-index
      • 定位的元素才有 z-index 值
      • 如果 z-index 相同, 后面的元素会压住前面的元素
      • z-index值没有单位，就是一个正整数。默认的z-index值是0
      • 从父现象：父亲怂了，儿子再牛逼也没用。意思是，如果父亲1比父亲2大，那么，即使儿子1比儿子2小，儿子1也能在最上层。

  2. 浮动

    • float

      • 浮动的特点：

        • 1.浮动的元素脱离标准流，不占原来的位置
        • 2.浮动的元素显示方式被改变
        • 3.浮动的元素以顶部对齐
        • 4.如果浮动的元素前面是标准流的元素，那么这个浮动的元素只会跟在标准流元素的后面,除非通过人为去改变才能让他的位置发生变化
        • 5.当有包含关系的时候，如果子元素浮动，不会跑出父元素，除非人为改动

      • 清除浮动 （清除浮动后造成的影响）

        1. clear：

          • left
          • right
          • both 清除左右两侧浮动的影响

        2. 额外标签法

          • 在浮动元素末尾添加一个空的标签

        3. overflow

          • 父级添加overflow属性，以触发BFC

        4. after伪元素

          • :after方式为空元素的升级版，好处是不用单独加标签了

          • tips

            • content:"." 里面尽量跟一个小点，或者其他，尽量不要为空，否则再firefox 7.0前的版本会有生成空格。

        5. before和after双伪元素清除浮动

          • 定义元素在不面对屏幕时是否可见。
      • 3d函数
        • matrix3d(n,n,n,n,n,n,n,n,n,n,n,n,n,n,n,n)
          • 定义 3D 转换，使用 16 个值的 4x4 矩阵。
        • translate3d(x,y,z)
          • 定义 3D 转化。
        • translateX(x)
          • 定义 3D 转化，仅使用用于 X 轴的值。
        • translateY(y)
          • 定义 3D 转化，仅使用用于 Y 轴的值。
        • translateZ(z)
          • 定义 3D 转化，仅使用用于 Z 轴的值。
        • scale3d(x,y,z)
          • 定义 3D 缩放转换。
        • scaleX(x)
          • 定义 3D 缩放转换，通过给定一个 X 轴的值。
        • scaleY(y)
          • 定义 3D 缩放转换，通过给定一个 Y 轴的值。
        • scaleZ(z)
          • 定义 3D 缩放转换，通过给定一个 Z 轴的值。
        • rotate3d(x,y,z,angle)
          • 定义 3D 旋转。
        • rotateX(angle)
          • 定义沿 X 轴的 3D 旋转。
        • rotateY(angle)
          • 定义沿 Y 轴的 3D 旋转。
        • rotateZ(angle)
          • 定义沿 Z 轴的 3D 旋转。
        • perspective(n)
          • 定义 3D 转换元素的透视视图。
