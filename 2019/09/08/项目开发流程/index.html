项目开发流程

  1. 确定开发需求(将客户的需求转换为我们自己的需求)出需求文档

  2. 原型图(由UI出)

  3. 技术选型（确定项目中所需要的具体的技术node vue和使用到的一些框架和库，约定规范）

  4. 分配任务开发项目

  5. 上线前的测试和bug修复

  6. 项目上线，客户验收

  7. 前端开发注意事项

    • 前端布局应适应各主流屏幕 (媒体查询, 百分比)
      • 标签的使用需注意 语义化
    • 需兼容主流浏览器/主流操作系统
      • 浏览器排行 https://www.mydll.com.cn/category-liulanqi-paiming
    • 输入框 类型 type
      • 如一个密码输入框 type 应为 password
    • 需注意 id 及 class 的命名
    • 用户体验
      • 应考虑用户习惯/方便用户进行操作
      • 删除/修改/支付 等操作需进行二次确认
    • 团体开发
      • 统一开发工具 换行方式 编码格式
      • 代码冲突需协商后合并
      • 一小改 一commit
      • id 及 class 命名约定
    • 未完.待续
