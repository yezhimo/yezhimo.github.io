  3. 标签语义化

    • 使用有意义的单词来作为标签的名字
    • 核心：在合适的地方使用合适的标签
    • 优点
      • 语义化HTML会使HTML结构变的清晰，有利于维护代码和添加样式
      • 让浏览器或是网络爬虫可以很好地解析，从而更好分析其中的内容
      • 便于团队开发和维护，语义化更具可读性，是下一步吧网页的重要动向，遵循W3C标准的团队都遵循这个标准，可以减少差异化。
    • 语义是否良好?
      • 当我们去掉CSS之后，网页结构依然组织有序，并且有良好的可读性

  4. 路径

    • 绝对路径
      • 网络绝对路径
        • 网络上一个位置
        • http://www.itcast.cn/images/logo.gif
      • 磁盘绝对路径
        • 磁盘上一个具体的位置
        • D:\web\img\logo.gif
    • 相对路径
      • 当前目录下
        • ./index.jpg
      • 当前目录下的另外一个目录(子目录)
        • ./imgs/index.jpg
      • 当前目录的父级目录下
        • ../src/fonts/woff.tff

  7. 标签属性

    • id

      • 作为元素的唯一标识(用于控制样式等)
      • 用于a标签进行锚链接跳转

    • class

      • 用以批量控制样式

    • style

      • 用以控制单个的元素样式

    • type

      值 描述
      button 定义可点击按钮（多数情况下，用于通过 JavaScript 启动脚本）。
      checkbox 定义复选框。
      file 定义输入字段和 “浏览”按钮，供文件上传。
      hidden 定义隐藏的输入字段。
      image 定义图像形式的提交按钮。
      password 定义密码字段。该字段中的字符被掩码。
      radio 定义单选按钮。
      reset 定义重置按钮。重置按钮会清除表单中的所有数据。
      submit 定义提交按钮。提交按钮会把表单数据发送到服务器。
      text 定义单行的输入字段，用户可在其中输入文本。默认宽度为 20 个字符。

    • disabled

      • disabled 属性规定应该禁用 input 元素。

    • accept

      • accept 属性只能与 <input type="file"> 配合使用。它规定能够通过文件上传进行提交的文件类型。
      • 请避免使用该属性。应该在服务器端验证文件上传。

    • control

      • controls 属性规定浏览器应该为视频提供播放控件。

    • autoplay

      • autoplay 属性规定一旦视频就绪马上开始播放。

    • url

      • 统一资源定位符（或称统一资源定位器/定位地址、URL地址等，英语：Uniform Resource Locator，常缩写为URL），有时也被俗称为网页地址（网址）。如同在网络上的门牌，是因特网上标准的资源的地址（Address）。

  8. html5中新增的标签

    • header:表示页面头部/页眉
    • nav：定义导航链接的部分
    • footer：定义文档或节的页脚
    • main：主体内容区域
    • section：区域块/section元素代表文档中的“节”或“段”
    • article: 标签规定独立的自包含内容
    • aside: 定义其所处内容之外的内容

  9. HTML5全局属性

    • data-* 使用 data-* 属性来嵌入自定义数据。
      • 属性名不应该包含任何大写字母，并且在前缀 “data-“ 之后必须有至少一个字符,属性值可以是任意字符串。
    • hidden
      • 它规定元素仍未或不再相关。浏览器不应显示已规定 hidden 属性的元素。hidden 属性也可用于防止用户查看元素，直到匹配某些条件（比如选择了某个复选框）。然后，JavaScript 可以删除 hidden 属性，以使此元素可见。
    • spellcheck
      • 规定是否对元素进行拼写和语法检查。
    • tabindex
      • abindex 属性规定元素的 tab 键控制次序（当 tab 键用于导航时）
      • 以下元素支持 tabindex 属性：<a>, <area>, <button>, <input>, <object>, <select> 以及 <textarea>。
    • contenteditable
      • 规定元素内容是否可编辑。
      • 可继承

  11. 快捷生成元素

    • 大于号形成一个层级关系

      • div>div>p

    • *号可以让标签多几次显示

      • ul>li*4

    • +号让后面的元素提高一级

      • div>ul>li*4+ol>li*4

    • ^ 号让后面的元素提高两级

      • div>ul>li*4^ol>li*4

  12. HTML5被删除的元素：

    • 能被css代替的标签：basefont、big、center、font、s、strike、tt、u。这些标签纯粹是为画面展示服务的，HTML5中提倡把画面展示性功能放在css中统一编辑。

    • 不再使用frame框架，HTML5不再支持，只支持iframe框架。

    • 只有部分浏览器支持的标签：applet、bgsound、blink、marquee等标签。

    • 其它被废除的标签：

      • 废除rb，使用ruby替代;

      • 废除acronym使用abbr替代;

      • 废除dir使用ul替代;

      • 废除isindex使用form与input相结合的方式替代;

      • 废除listing使用pre替代;

      • 废除xmp使用code替代;

      • 废除nextid使用guids;

      • 废除plaintex使用“text/plian”（无格式正文）MIME类型替代。
